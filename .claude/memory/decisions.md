# Decisions Memory

Use this file to record durable project decisions whose rationale would help future sessions.

## What belongs here

- Long-lived technical decisions with meaningful trade-offs
- Decisions whose rationale is easy to lose if only the code remains
- Constraints that should shape future suggestions

## What does not belong here

- Plain architecture descriptions
- Temporary implementation plans
- Historical details that are obvious from git history alone
- Speculation about future work

## Entry format

### YYYY-MM-DD — Decision title
- **Decision:** What was decided
- **Why:** Why this path was chosen
- **How to apply:** How future work should respect this decision
