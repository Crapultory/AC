# AISOC MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a new `hermes aisoc` command that serves an AISOC-branded web console with login, embedded chat-tui, sessions relaunch, cron, skills, memory editing, and logs on port `9120`.

**Architecture:** AISOC ships as a standalone local-first web product with a dedicated FastAPI backend under `aisoc/backend/` and a dedicated React frontend under `aisoc/frontend/`. The backend reuses proven Hermes primitives for PTY chat, sessions, cron, skills, and logs through thin service adapters, while the frontend uses an authenticated shell with a login page and protected routes instead of Dashboard's injected session token model.

**Tech Stack:** Python, FastAPI, Uvicorn, pytest, React 19, TypeScript, Vite, React Router, xterm.js, Vitest, Testing Library.

## Implementation Notes

- Reuse existing Hermes code by extracting or wrapping helpers, not by importing Dashboard React pages directly.
- Keep chat as an embedded PTY experience. Do not rebuild the transcript or composer in React.
- Use `Authorization: Bearer <token>` for protected REST APIs. Use `?token=<token>` on websocket URLs because browser websocket upgrades cannot set arbitrary auth headers.
- Store the browser token in `localStorage` under a single AISOC-specific key such as `aisoc.sessionToken`.
- Resolve `SOUL.md` from `get_hermes_home() / "SOUL.md"` and built-in memory files from `get_hermes_home() / "memories"`.
- Preserve Dashboard's localhost safety defaults and require `--insecure` for non-loopback binding.

class AisocSettings:
    host: str

    token_source: Literal["env", "generated"]

def verify_bearer_token(request: Request) -> None:
    header = request.headers.get("Authorization", "")
    if header != f"Bearer {settings.session_token}":
        raise HTTPException(status_code=401, detail="Unauthorized")

def verify_ws_token(token: str | None) -> None:
    if token != settings.session_token:
